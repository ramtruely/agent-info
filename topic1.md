Create a topic that helps employees request GitHub repository access.

The conversation should:

1. Greet the user.

2. Ask for the employee's full name.

3. Ask for employee ID.

4. Ask for company email address.

5. Ask for the GitHub repository name.

6. Ask what level of access is required with three options:
- Read
- Write
- Admin

7. Ask the reason for requesting access.

8. Display a formatted summary of all collected information.

9. Ask the user to confirm whether the information is correct using Yes or No.

10. If Yes, display:
"Your GitHub access request has been submitted successfully. The DevOps team will review your request."

11. If No, display:
"Request cancelled. Please start again if you want to submit a new request."

Use best practices by storing every response in variables and creating a clean conversational flow.
